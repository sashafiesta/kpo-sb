# Возможности

- Добавление / удаление животных
- Добавление / удаление предметов
- Вывод отчета по животным / предметам
- Вывод списка животных для контакта с посетителями
- Расчет потребления еды
- Автоматический учет инвентарных номеров

# Архетектура

Есть единый абстрактный класс Animal, от которого наследуются классы Herbo и Predator.
От них дальше наследуются непосредственно животные.

Есть единый интерфейс InventoryItem, от которого наследуются непосредственно предметы.

Все элементы распределены в соответствующие пакеты

Для сервисов реализованы интерфейсы.
    
DI реализован кастомный в com.sashafiesta.zoo.di

Unit-тесты реализованы при помощи **junit**

Взаимодействие с программой сделано в режиме терминала.

# SOLID

## Single Responsibility:
Каждый класс отвечает за свою задачу. (IZoo / ZooImpl, например, содержит только необходимые методы и поля) 

## Open/Closed
Новые объекты можно добавлять, наследуюясь от соответствующих абстрактных классов. Дополнительных изменений не требуется (и они недопустимы).

## Liskov Substitution
Интерфейсы, IAlive и IInventory можно использовать вместо классов, которые их реализуют.

## Interface Segregation
Обеспечено разделение интерфейсов: IAlive и IInventory.

## Dependency Inversion
Все зависит не от ZooImpl, а от абстракции IZoo, внедренной при помощи DI.
